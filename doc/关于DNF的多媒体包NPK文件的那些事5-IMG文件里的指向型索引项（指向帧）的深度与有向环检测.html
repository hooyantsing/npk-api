本文介绍了DNF多媒体包NPK中的IMG文件中指向型索引项（指向帧）的工作原理和递归算法在检测有向环中的应用。当遇到指向帧时，DNF会按Link深度查找目标图片帧，但如果存在有向环，会导致程序错误并闪退。为了避免这种情况，提出了一个最大检测深度N的递归算法来判断帧的有效性。实验结果显示，通过调整最大检测深度，可以确认IMG文件中是否存在有向环并确定最大Link深度不超过2。

IMG里的指向型索引项（指向帧）的描述和用途已经在前文讲过。

当贴图读取到指向帧时，会顺着该索引项的第二字节（linkTo）来寻找它所指向的其他帧，若该帧也是指向帧，则会继续寻找，直至找到图片帧。如下图所示，第3、4帧的Link深度为1，直接指向第2帧；第6帧Link深度也为1，直接指向第5帧；而第8帧的Link深度为2，因为它指向了Link深度为1的第6帧，而最终指向了第5帧：


很明显，DNF在读取到指定帧号是，需要经过Link深度次查找，才能找到目标图片帧。但有时候，会存在多个指向帧的指向方位构成一个有向环，如下图的2、3、4帧；或者更简单的一种特例，即指向帧指向自己，如下图的第5帧，很明显他们的Link的深度均为∞：


由于DNF自身不具备判断指向帧所指向的帧的类型，也不具备直接计算每个帧号的Link深度的能力，因此当DNF读取如上图所示的2、3、4、5帧时，则会不断地寻找下去，直至暴栈，其直接结果就是DNF出现错误并且闪退。因此任何一个IMG文件里的指向帧应避免构成一个有向环，或者更直接的，避免文件里出现指向帧指向指向帧的情形，而实际上，DNF官方包含的IMG文件Link的深度最大值为2。

检测一个IMG文件里是否存在有向环，需要读取这个IMG内所有帧的帧类型以及作为指向帧时的指向帧号，然后确定最大检测深度N。其中最大检测深度N是一个正整数，作为该帧是否为有效帧的一个标的，该算法可以采用递归。其中当最大检测深度N=0时，始终返回无效帧，即-1；最大检测深度N>0时，当帧为图片帧时，该帧为有效帧，并返回自己的帧号；若该帧为指向帧，则返回N-1时，该帧的指向帧号对应的帧的结果（尤其地，当最大检测深度为-1时，该帧若为指向帧则返回无效帧）。其相关代码如下（因为有KOISHIAPI支持所以当伪码看一下吧）： 


i32 IMGobject::linkFind(b32 pos, b32 depth){
	PICinfo pi;
	PICInfo(pos, pi);
	if(depth == 0){
		return -1;
	}else{
		if(pi.get_format() == LINK){
			return linkFind(pi.get_linkTo(), depth-1);
		}else{
			return pos;
		}
	}
}


因此可以看出，若IMG文件里存在有向环，则最大检测深度N无论取多高，有向环内的帧号均会返回-1。

下面伪代码计算从女魔法师皮肤文件的提取第一个IMG文件进行试验： 

using namespace Koishi;
int _tmain(int argc, _TCHAR* argv[])
{
	NPKobject no;
	no.loadFile("skin.npk");
	IMGobject io;
	no.IMGextract(0, io);
	int *s = new int[io.count];
	printf("检测深度：1\n");
	for(int i = 0;i<io.count;i++){
		s[i] = io.linkFind(i, 1);
	}
	disp(s, io.count);
	printf("\n检测深度：2\n");
	for(int i = 0;i<io.count;i++){
		s[i] = io.linkFind(i, 2);
	}
	disp(s, io.count);
	system("pause");
	return 0;
}
